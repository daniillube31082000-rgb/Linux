В мире Linux есть выражение Всё — это файл, которое отражает один из принципов системы: самые разные объекты — программы, настройки, устройства — представлены как **файлы.** Но при этом существует несколько типов таких файлов.

### **1. Обычные файлы**
+ **Самые привычные:** тексты, картинки, документы, программы и так далее.
+ **Нет строгих правил насчёт расширений:** если видите файл .txt или .pdf, это лишь подсказка для пользователя, а не признак типа для системы.
### **2. Каталоги (папки)**
+ **В Linux их называют директориями.** Внутри хранятся другие файлы и каталоги.  

+ **Важная особенность:** у них есть свои права доступа (кто может войти, кто может смотреть содержимое и т.д.).

### **3. Ссылки**
+ **Символические ссылки:** похожи на ярлыки в Windows. Указывают путь к другому файлу или каталогу.
+ **Жёсткие ссылки:** несколько разных имён указывают на один и тот же объект. Если изменить файл через одну ссылку, увидите то же самое через другую.  

**Особенности жёстких ссылок:**
- Работают только в пределах одной файловой системы.
- Имеют тот же inode, что и исходный файл.
- Остаются рабочими даже после удаления исходного файла.
- Как правило, не создаются для каталогов (кроме системных служебных случаев).  

**Важно помнить разницу:**

1. Если удалить оригинальный файл, на который указывает **символическая ссылка**, она «сломается» и перестанет работать.
2. Если удалить файл, к которому ведёт **жёсткая ссылка**, данные останутся доступными через другие имена, потому что все жёсткие ссылки равноправны.
```
# Создать символическую ссылку
ln -s /путь/к/оригиналу /путь/к/ссылке
```
```
# Создать жёсткую ссылку
ln /путь/к/оригиналу /путь/к/ссылке
```
```
# Посмотреть список файлов и их типы
ls -l
```
В выводе команды ***ls -l*** символическая ссылка будет отмечена буквой ***l*** в начале строки и стрелкой (->) перед оригинальным файлом. Жёсткая ссылка в выводе будет выглядеть как обычный файл, но в колонке с количеством ссылок будет число больше 1.  

### **4. Устройства**
+ **Файлы устройств:** Linux обращается к оборудованию (диск, клавиатура, USB) как к обычным файлам.
+ **/dev:** специальная папка, где находятся файлы устройств. Такие файлы обеспечивают доступ к драйверам оборудования. Через них система и программы могут работать с дисками, сетевыми картами, принтерами и другим «железом» так, как будто это обычные файлы.  

### **5. Специальные каналы (FIFO) и сокеты**
+ **FIFO (именованный канал):** способ передавать данные между программами через файл.
+ **Сокеты: файлы,** которые программы используют для общения друг с другом (иногда вместо сети).  

 **Итог:** хотя мы называем всё в Linux файлами, они могут сильно отличаться по назначению. Обычные файлы — это то, с чем вы работаете постоянно (документы, видео, программы). Каталоги — папки. Ссылки — указатели. Устройства и специальные каналы — тоже представлены как файлы, но служат для взаимодействия с реальным оборудованием и межпроцессным обменом данными.

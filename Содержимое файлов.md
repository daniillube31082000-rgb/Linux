Когда вы работаете в терминале, порой нужно быстро заглянуть в файл, чтобы понять, что он содержит. Это может быть текстовая заметка, конфигурационный файл или лог-сообщения. Ниже рассмотрим ключевые команды, которые помогут подсмотреть содержимое без необходимости открывать графический редактор.

**1. cat**
+ **Объединение (concatenate) и вывод.** В простейшем варианте выводит содержимое одного файла.  

+ **Пример:**  

    ```cat myfile.txt```

                  
Терминал покажет весь текст из ``myfile.txt``  

**Недостаток:** если файл большой, вывод пробежит слишком быстро. Для длинных файлов удобнее использовать постраничный просмотр.  

**2. less**  

+ **Интерактивный постраничный просмотр.** Позволяет удобно листать содержимое.
+ **Пример:**  

    ```less largefile.log```

                  
Вы можете:  

+ Прокручивать колёсиком мыши, стрелками вверх/вниз или PgUp/PgDn.
+ Нажать /, ввести слово и нажать Enter — программа найдёт это слово в тексте.
+ Выйти из режима просмотра, нажав ``Q``.
+ Отлично подходит для чтения длинных файлов, логов и больших конфигураций.  

**3. more**  

+ **Похоже на less**, но с чуть более ограниченным функционалом.
+ В большинстве современных систем лучше использовать ``less``, так как он умнее (например, позволяет прокрутку вверх).
+ Однако встретить ``more`` всё ещё можно в некоторых минималистичных окружениях.  

**4. head и tail**  

+ ``head`` показывает верх файла (по умолчанию 10 строк).  

**Полезные флаги команды head:**
+ ``-n <число>`` — вывести указанное количество строк с начала файла (по умолчанию 10).
+ ``-q — "quiet"`` — подавляет вывод имени файла в случае, если выводится несколько файлов.
+ ``-v — "verbose"`` — всегда показывает имя файла (обратное q).
+ ``-r — "reverse"`` — выводит строки в обратном порядке (поддерживается не во всех версиях head).
+ ``-f ``— работает аналогично ``tail -f``, позволяя "следить" за обновлениями в начале файла (редко используется).

+ ``tail`` показывает низ файла (по умолчанию 10 строк).
+ **Опции:**  

+ ``head -n 20 myfile.txt`` — вывести первые 20 строк.
+ ``tail -n 20 myfile.txt`` — вывести последние 20 строк.
+ ``tail -f logfile.log`` — режим следить: новые записи будут выводиться в реальном времени, как только появляются в файле (удобно для логов).
Иногда нужно вывести строки не с начала и не с конца файла, а из середины.
Для этого можно комбинировать команды ``head`` и ``tail``.  

    ``head -n 15 example.txt | tail -n +5``

                  
``Эта команда сначала берёт первые 15 строк файла, а затем выводит строки, начиная с 5-й. В итоге на экране будут строки с 5-й по 15-ю.``

+ **Пример:**  
``head some_file.txt``  
``tail -f /var/log/syslog ``

                  
Первая команда покажет начало ``some_file.txt``, а вторая — будет **в прямом эфире отображать новые строки** в системном логе.  

**5. uniq**  

+ ``uniq`` — выводит только уникальные строки из входных данных, но сравнивает только соседние строки.
Если одинаковые строки находятся в разных местах файла, команда их не уберёт.

+ Чтобы удалить **все** дубликаты, файл сначала сортируют:

``sort duplicates.txt | uniq``

                  
Более короткий вариант — использовать флаг ``-u`` у ``sort``, который сразу удаляет дубликаты:

``sort -u duplicates.txt``

                  
+ **Пример:**  


Было:

``apple``  

``orange``  

``apple``  

``banana``  

``orange``

                  
После ``sort -u duplicates.txt``:

``apple``  

``banana``  

``orange``  


                  
Чтобы подсчитать количество строк в файле, используйте:

``wc -l data.txt``

                  
Это покажет число строк и имя файла.

Важно: ``uniq`` сравнивает только соседние строки. Чтобы удалить все дубликаты, файл сначала сортируют:

``sort duplicates.txt | uniq``

                  
Или используют короткий вариант:

``sort -u duplicates.txt``

                  
**5. file**

+ **Не показывает содержимое**, но определяет тип файла.  

+ **Пример:**  

``file myfile.txt``  

``file image.png``

                  
Вы увидите, является ли ``myfile.txt`` текстовым файлом, а ``image.png`` — PNG-изображением и т. д.  

+ Полезно, когда вы не уверены, действительно ли файл текстовый.  

+ ### Примеры применения
1. **Проверка логов:**  

    ``less /var/log/syslog``

                  
Листаете вверх-вниз, ищете нужные ключевые слова и потом нажимаете ``Q`` для выхода.  

2. **Смотреть новые записи в реальном времени:**  

    ``tail -f /var/log/nginx/access.log``

                  
Полезно, если вы хотите отслеживать, кто заходит на ваш веб-сайт в текущий момент.  

3. **Быстрая проверка что за файл?:**  

    ``file unknown.bin``

                  
Может оказаться, что это текстовый скрипт или архив.
### **Итог**  

+ ``cat`` — самый простой способ показать содержимое, но подходит для небольших файлов.
+ ``less`` — интерактивный просмотр, удобен для длинных текстов и логов.
+ ``head / tail`` — вывести начало/конец файла, можно следить за изменениями с опцией ``-f``.
+ ``file`` — определяет тип файла, если вы не уверены в его формате.

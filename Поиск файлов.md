Рано или поздно вам понадобится найти в системе какой-то файл или группу файлов, причём неизвестно, где именно они лежат. В Linux есть несколько способов поиска, и самый гибкий — это команда **find.** В первой части мы рассмотрим основы работы с ней.

### **Синтаксис команды find**  

``find [где искать] [условия] [действия]``

                  
+ **Где искать** — каталог, в котором начинать поиск (например, ``/home/user или /etc``, а можно просто ``/`` для поиска по всей системе).
+ **Условия (options)** — это критерии, по которым выбираются файлы (имя, тип, дата изменения и т. д.).
+ **Действия (options)** — что делать с найденными файлами (выводить на экран, удалять, копировать и т. д.).
Простой пример: поиск по имени
+ ``find /home -name "notes.txt"``
+ Искать в каталоге ``/home`` (включая все подкаталоги) файл с именем ``notes.txt``.
+ **-name** — указывает, что ищем по имени.
+ Кавычки нужны, если в шаблоне присутствуют спецсимволы (*, ?, [ ]), но даже для простых имён это бывает полезно для привычки.
### **Использование шаблонов (wildcards)**
+ ``*`` — означает любое количество любых символов.
+ ``?``— один любой символ.
+ **Примеры:**  

    ``find /home -name "*.txt"``  

    ``find /var/log -name "syslog.*"``

                  
+ ``*.txt`` — все файлы, оканчивающиеся на ``.txt.``
+ ``syslog.*`` — файлы, начинающиеся на ``syslog.`` с любым продолжением (например, ``syslog.1``).
+ ``[]`` — класс символов: например, ``file[0-9].txt`` совпадает с ``file1.txt``, ``file2.txt`` и т. п.

Важно: эти шаблоны — это glob-маски (*, ?, []). Символы вроде $ относятся к регулярным выражениям и работают с -regex, а не с -name.

### **Поиск по типу**
+ ``-type f`` — искать только обычные файлы (``f = file``).
+ ``-type d``— искать только каталоги (``d = directory``).
+ **Пример:**  

    ``find /home -type d -name "docs*"``

                  
Будет искать каталоги, чьё имя начинается на ``docs``.
+ ``-size +10M``— файлы больше 10 мегабайт (есть суффиксы ``k, M, G``).

+ ``-empty`` — пустые файлы/каталоги (совместите с ``-type f``, чтобы искать только файлы).

+ Временные критерии:

+ ``-mtime -7``— изменённые за последние 7 дней;

+ ``-mmin -60`` — за последние 60 минут;

+ ``-newermt "7 days ago"`` — альтернативный человеко-читаемый формат.

### **Параметр -print**
+ По умолчанию ``find`` выводит найденные пути на экран. Это равносильно флагу ``-print``.
+ При желании можно явно указать ``-print`` или комбинировать с другими действиями (например, -exec, но об этом в следующей части).
### **Подсказки**
+ **Регистрозависимость.** Опция -name ищет с учётом регистра. Если нужно игнорировать регистр, используйте ``-iname``.
+ **Права доступа.** Если вы ищете в каталогах, где у вас нет прав на чтение, find выведет ошибку ``Permission denied``. Иногда приходится запускать команду от root или использовать ``sudo``.
+ **Остановка поиска.** Если запустили поиск по всему диску, а оно слишком долго, можно прервать процесс клавишами ``Ctrl + C``.
+ Кавычки вокруг масок помогают избежать подстановки оболочкой: пишите -name ``"*.txt", а не -name *.txt``.
### **Итог**
+ ``find [путь]`` ``-name "шаблон"`` — базовый поиск по имени в указанной директории.
+ ``*`` и ``?`` позволяют использовать шаблоны для более гибкого поиска.
+ ``-type`` даёт возможность искать только файлы (``-type f``) или только каталоги (``-type d``).
+ ``find`` — очень мощная команда, позволяющая задать и другие критерии (размер, время модификации, права доступа). Во второй части мы рассмотрим более продвинутые примеры и опцию ``-exec``   .
